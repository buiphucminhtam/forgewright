## 2026-06-16 — Custom / Maintenance (GitNexus)
### Blocker Type: Tooling / External
### Problem: `gitnexus analyze` fails with `Binder exception: Trying to insert into an index on table File but its extension is not loaded.` when trying to incrementally update existing indexes.
### Failed Attempts: Running `gitnexus analyze` without environment variables.
### Research Source: DuckDB/LadybugDB documentation on FTS index limitations and lazy-loading in sandboxed/strict environments.
### Solution: Run the analysis by prefixing it with the environment variable `GITNEXUS_LBUG_EXTENSION_INSTALL=auto` (e.g., `GITNEXUS_LBUG_EXTENSION_INSTALL=auto gitnexus analyze <path>`).
### Key Insight: When DuckDB/LadybugDB encounters existing FTS indexes but the FTS extension is not pre-installed or loaded, dynamic extension loading must be enabled explicitly using `GITNEXUS_LBUG_EXTENSION_INSTALL=auto`.
### Apply When: Performing `gitnexus analyze` or re-indexing in environments where the index file already contains FTS indices.
### Planning Review Trigger:
### - Planning Gap: Did not anticipate that the new gitnexus version (v1.6.7) or existing database format would require explicit FTS extension installation permissions in this shell environment.
### - Planning Improvement: Always check capabilities using `gitnexus doctor` or use `GITNEXUS_LBUG_EXTENSION_INSTALL=auto` by default when automating indexing.
